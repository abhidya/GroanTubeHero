# Current State Audit - Groan Tube Hero

This document audits the legacy (V1) game structure, active Roblox Studio instances, and network APIs to establish a baseline for the `WorldV2` circular rebuild.

---

## 1. Active Workspace & World Roots
The current game environment consists of three primary root containers:
- **`workspace.Stage`**: Contains all stage geometry (such as the perform deck), speaker parts, and prototype kiosks.
- **`workspace.TourBus`**: Contains the static tour bus mesh model.
- **`workspace.Unused_MapAssets`**: Holds imported package templates and character models used for spawning the horde.

---

## 2. Legacy Objects Designated for Archival
To clean up the workspace, all autogenerated, rectangular, and placeholder parts are marked for archival. They will be parented to `workspace.__WorldArchive_DoNotDelete` (or `ServerStorage.WorldArchive`) and disabled:
- **`workspace.Stage.ArenaFloor`**: Generated rectangular floor part.
- **`workspace.Stage.LavaMoat`**: Leg-killing gray/red placeholder box beneath the stage.
- **`workspace.Stage.StageTopGlow`**: Procedural neon indicator plate.
- **`workspace.Stage.HordeLane`**: Single rectangular lane walkway.
- **`workspace.Stage.BrainrotClimbRamp`**: Blocky climbing ramp mesh.
- **`workspace.Stage.StoreKiosk` / `UpgradeKiosk` / `MissionBoard`**: Flat block representations of vendor stations.
- **`workspace.Stage.SpeakerStacks`**: Default gray thumping blocks.
- **`workspace.Stage.AudienceZone`**: Forcefield cylinder enclosing spectator dummies.
- **`workspace.Stage.CleanSigns`**: Duplicate signs causing billboard label overlaps.
- **`workspace.Stage.Spotlights`**: Unanchored light pole parts.

---

## 3. UI/Menu Hierarchy Audit
Active screen interfaces reside under `StarterGui` (cloned to `PlayerGui` at runtime):
- **`StarterGui.RhythmHUD`**: Contains the dynamic gameplay note highway.
- **`StarterGui.NavigationMenu`**: The side navigation bar (contains buttons: Choose Song, Store, Upgrades, Missions, Tutorial, Security).
- **`StarterGui.SongSelect`**: Modal list for choosing songs.
- **`StarterGui.StoreGui`**: Shop interface for buying skins and tubes.
- **`StarterGui.UpgradeGui`**: Interface for buying speaker and fence stability boosts.
- **`StarterGui.MissionGui`**: Display panel for challenges.

---

## 4. Network Remotes & Event Mapping
Communication between client and server is handled via `ReplicatedStorage.Remotes`:
- **`StartSong`** (RemoteFunction): Invoked by client to start a chart session.
- **`NoteJudged`** (RemoteEvent): Server broadcasts scoring judgements to clients.
- **`HordeUpdate`** (RemoteEvent): Server broadcasts horde position updates.
- **`SongComplete`** (RemoteEvent): Server signals chart completion.

## 6. Known Codebase Bugs Identified
1.  **Latency Judgment Defect (`AntiExploitService.lua`)**: `ValidateNoteHit` was comparing server offset values directly without reading or validating the client's network timing (`clientDelta`), leading to latency-induced note misses.
2.  **Stale Anchors in Spawner (`GameBootstrap.server.lua`)**: The `ensurePart` subroutine was returning early if a part already existed, which meant updated CFrames, sizes, and physics parameters were ignored.
3.  **Overlapping Billboards (`GameBootstrap.server.lua`)**: The `createBillboard` routine was hardcoded with `AlwaysOnTop = true`, making billboard text draw over physical stage geometry and causing unreadable visual overlapping.
4.  **Horde Height Offset Bug (`HordeClient.client.lua`)**: `ensureStageHorde` was overwriting server-defined spawns at runtime, causing cloned horde monsters to walk underneath the stage.
5.  **Navigation Modal Trap (`RhythmClient.client.lua`)**: Closing the Song Select modal via the "X" button did not restore the NavigationMenu properly, leaving players stuck in the lobby.
